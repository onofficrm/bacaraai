---

## 1. 1차 목표

1. DB 실시간 결과와 UI 로드맵/통계를 **현재 슈 기준으로 일치**
2. GPT · Claude · Gemini API를 관리자에서 등록하고 **실제 호출**
3. 서버 통계 + 멀티 LLM + 결정 엔진으로 **다음 판 의견** 산출
4. 오토베팅 **「AI 추천대로」** 모드에서 조건 충족 시 **시스템이 자동 베팅**
5. 사용자 도움: **위험 코치 · 판단 설명 · 세션 복기**

---

## 2. 핵심 전제 (1차에서 고정)

- 바카라 각 판은 **거의 독립**이다. 패턴은 설명용이며 **지속적 예측 우위를 보장하지 않는다.**
- LLM에게 **숫자/표본 계산을 맡기지 않는다.** 서버가 통계를 계산하고 AI는 해석·관망 판단만 한다.
- AI가 **금액·손절을 임의 변경하지 않는다.** 금액은 오토베팅 설정, 차단은 윈컷/로스컷 규칙.
- 브라우저 2초 폴링마다 AI를 호출하지 **않는다.** **새 결과 ID당 1회** + 서버 캐시.

---

## 3. 처리 흐름 (1차)

```
DB 새 결과 (bacaraai)
    ↓
live_results.php → 현재 슈(game_no 리셋 이후)만 반환
    ↓
프론트 useLiveTable 폴링
    ↓
결과 ID 변경 시 ai_analyze.php (캐시 miss면)
    ↓
서버 패턴 통계 계산
    ↓
GPT / Claude / Gemini 병렬 호출 (curl_multi)
    ↓
결정 엔진 → finalOpinion + auto_bet_allowed
    ↓
g5_bacara_ai_prediction 저장 · 다음 결과로 적중 정산
    ↓
UI 표시 / (조건 시) 오토 라이브 자동 베팅
```

---

## 4. 데이터 · 슈 정의

| 항목 | 내용 |
|------|------|
| 소스 테이블 | `bacaraai.bacaraai` (live config DB) |
| 계정 | live config `account` → 로그인 ID → `awesome` → 테이블 fallback |
| `game_no` | **회차 카운터** (1…N 후 리셋). 슈 ID가 아님 |
| 현재 슈 | 마지막 `game_no = 1` 이후 `id` 오름차순 (+ game_no 감소 시 trim) |
| 라이브 테이블 | TABLE1 = `MD2729` (그 외는 MOCK) |

**버그 수정(1차):** 예전에는 최신 `game_no`로 필터해 과거 슈의 같은 회차가 섞였음 → 슈 구간 조회로 교체.

## 5. API · 설정

### 관리자

- 경로: 회원관리 → **AI API 설정**  
  `/plugin/bacara_wallet/admin/ai_keys.php`
- 저장: `data/bacaraai-ai.config.php` (Git 제외)
- 항목: OpenAI / Anthropic / Gemini 키·모델, 분석 ON/OFF, **AI 자동 베팅 허용**

### 런타임 API

| API | 역할 |
|-----|------|
| `live_results.php` | 현재 슈 결과 JSON |
| `ai_analyze.php` | 통계 + 3LLM + 결정 + 캐시 |
| `ai_session_review.php` | 세션 종료 복기 |

### 예측 저장

- 테이블: `g5_bacara_ai_prediction`
- 키: `(table_name, source_result_id)` UNIQUE
- 다음 결과로 `actual_result` / `hit` 정산 (Tie·WAIT는 적중 제외)

## 6. 결정 엔진 (1차 임계값)

### 화면 추천 (finalOpinion)

| 조건 | 값 |
|------|-----|
| 다수결 | ≥ 2/3 (PLAYER/BANKER) |
| 평균 신뢰도 | ≥ 55% |
| 패턴 표본 | ≥ 20 |
| P/B 격차 | ≥ 8% |
| 미충족 시 | WAIT |

### 자동 베팅 (auto_bet_allowed)

관리자 **AI 자동 베팅 ON** + 아래 **전부** 충족:

| 조건 | 값 |
|------|-----|
| API 키 | 3개 모두 등록 |
| 호출 | 3개 모두 정상(에러 없음) |
| 다수결 | ≥ 2/3 |
| 신뢰도 | ≥ 65% |
| 표본 | ≥ 30 |
| P/B 격차 | ≥ 10% |
| 세션 | **라이브** |
| 전략 | **AI 추천대로** |
| 베팅창 | 잔여 **≥ 5초** |
| 금액 | 세션 초기/마틴 설정 (AI 미결정) |

미충족이면 화면에는 참고 추천만, 자동 베팅 안 함.

## 7. 오토베팅 전략 역할 분담

| 전략 | 진입 신호 | 금액 |
|------|-----------|------|
| **AI 추천대로** | `auto_bet_allowed` + PLAYER/BANKER | 세션 금액 플랜 |
| **내가 만든 패턴** | `PatternCase` 매칭 (BB→P 등) | shared / per_case |

패턴 기본 케이스(참고): BB→P, PP→P, BBBB+ 후 P→B.

## 8. 사용자 도움 기능 (1차)

### 위험 코치

- 위치: 세션 바
- 트리거: 로스컷/윈컷 근접, 마틴 상승, 연패, 오토 성과 저조
- 파일: `builder/ bacaraai_system/src/utils/riskCoach.ts`

### AI 판단 설명

- 테이블 카드: `appliedRule` 한 줄
- 우측 패널: 결정 근거 + GPT/Claude/Gemini 의견·신뢰도·이유 1줄

### 세션 복기

- 윈컷/로스컷/수동 종료 모달
- 서버 통계 + (가능 시) LLM JSON 요약·다음 제안
- API: `ai_session_review.php`
